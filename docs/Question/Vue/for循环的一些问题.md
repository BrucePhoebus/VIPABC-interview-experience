# for循环的一些问题

## for循环为什么不好用下标作为key

首先，如果往前面插入或删去一条，更新的时候，则其他部分也会受到影响，因为key会重新计算，diff算法会比对虚拟DOM节点内容是否发生变化，最后发现key不正确了，所以一批都需要更新

所以最好使用的key是不会变化的，这样每次插入删除都不会导致原有一些节点发生变化，毕竟key不变、其余属性也不变，diff算法就不会去更新这些内容，效率就提高了
